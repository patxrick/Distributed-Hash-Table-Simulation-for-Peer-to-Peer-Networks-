# Distributed Hash Table (DHT) Simulation for Peer-to-Peer Networks

## 📌 Overview

This project presents a simulation of a Distributed Hash Table (DHT) based Peer-to-Peer (P2P) Network using Python. The primary objective is to demonstrate decentralized data storage, efficient key-value lookup, node communication, and scalable routing mechanisms used in modern distributed systems.

The project models how peers interact in a decentralized environment without relying on a centralized server, making it highly relevant for distributed computing and peer-to-peer architectures.

---

## 🚀 Key Concepts

### 🔹 Distributed Hash Table (DHT)

A Distributed Hash Table is a decentralized distributed system that provides efficient lookup services similar to a hash table. Data is distributed across multiple nodes, and each node is responsible for storing a portion of the data.

Key features include:

* Decentralized architecture
* Efficient key-based lookup
* Scalable node management
* Fault-tolerant data distribution

---

### 🔹 Peer-to-Peer Networks

Peer-to-Peer (P2P) systems allow nodes (peers) to communicate directly without centralized coordination. Each peer can act as both a client and server, improving scalability and reliability.
